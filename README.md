# 门店报修派工与回访工作台

一个完整的门店报修工单管理系统，支持从门店提交报修、调度分配工程师、工程师维修处理、到质检回访关单的完整业务流程。

## 功能特性

- **角色分离**：门店、调度、工程师、质检四个独立角色，各司其职
- **流程可控**：严格的工单状态流转，防止越权操作
- **可追溯**：完整的操作历史时间线，记录每次派工、退回、回访的详情
- **防错设计**：内置失败路径处理，阻止非法操作
- **数据持久化**：工单状态、筛选条件、导出结果在刷新后保持

## 技术栈

- React 18 + TypeScript
- Vite 构建工具
- TailwindCSS 样式框架
- Zustand 状态管理
- Lucide React 图标库
- date-fns 日期处理

## 演示流程

### 准备工作

1. 启动应用后，默认以**门店**身份登录
2. 点击右上角的**重置**按钮可以恢复初始演示数据

### 主流程演示

#### 第一步：门店提交报修

1. 确保当前角色为**门店**
2. 点击右上角**新建报修**按钮
3. 选择门店（以朝阳店为例）
4. 填写设备类型：`中央空调`
5. 填写故障描述：`空调不制冷，室内温度降不下来`
6. 点击**提交报修**
7. 自动跳转回工作台，新工单已生成

#### 第二步：调度派工

1. 在顶部切换角色为**调度**
2. 在工单列表中找到刚才创建的工单（状态：待派工）
3. 点击工单打开详情
4. 在派工面板选择工程师（选择空闲的工程师）
5. 点击**确认派工**
6. 工单状态变为**已派工**，分配给对应工程师

#### 第三步：工程师接单并维修

1. 切换角色为**工程师**
2. 在工单列表中找到分配给自己的工单（状态：已派工）
3. 点击工单打开详情
4. 点击**接单**按钮
5. 工单状态变为**维修中**
6. 在维修记录表单中填写维修过程
7. 点击**保存记录**
8. 点击**提交完工**
9. 工单状态变为**待回访**

#### 第四步：质检回访

1. 切换角色为**质检**
2. 在工单列表中找到待回访的工单
3. 点击工单打开详情
4. 选择评价（满意/基本满意/不满意）
5. 填写回访备注
6. 点击**确认回访**
7. 如果选择满意/基本满意，工单状态变为**已完成**
8. 如果选择不满意，工单状态退回为**维修中**

### 失败路径演示

#### 并发抢派

1. 保持当前为**调度**角色
2. 找到状态为**待派工**的样例工单 `WO-2024-0001`
3. 点击派工并选择一个工程师
4. 派工成功后，工单状态变为**已派工**
5. 再次尝试派工同一张工单
6. 系统会提示：`该工单已被其他调度派工`

#### 提前关单拦截

1. 切换到**工程师**角色
2. 找到状态为**维修中**的工单
3. 注意：工程师无法直接关单，只能提交完工
4. 完工后工单进入**待回访**状态
5. 即使工程师在线，也无法绕过质检关单

#### 已撤销工单重开

1. 切换到**门店**角色
2. 找到状态为**已撤销**的工单 `WO-2024-0004`
3. 尝试对该工单进行任何操作
4. 系统会提示：`该工单已撤销，无法再次操作`
5. 如需重新处理，需由门店提交新的报修

| 派工 | ❌ | ✅ | ❌ | ❌ |
| 接单 | ❌ | ❌ | ✅ | ❌ |
| 维修记录 | ❌ | ❌ | ✅ | ❌ |
| 完工提交 | ❌ | ❌ | ✅ | ❌ |

## 数据持久化

所有数据存储在浏览器 localStorage 中，包括：

- 工单数据（状态、维修记录、回访备注）
- 筛选条件
- 当前角色
- 导出结果

刷新页面后数据自动恢复。如需重置数据，点击右上角重置按钮。
